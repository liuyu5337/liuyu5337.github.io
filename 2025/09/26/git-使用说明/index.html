使用

如果ssh key在生成的时候，设置了密码,每次git需要输入密码，为了避免每次输入密码，可以使用 eval "$(ssh-agent -s)",主要作用是管理 SSH 密钥的密码缓存，避免频繁输入密钥密码。

为了避免依赖 ssh-agent，推荐通过 ~/.ssh/config 明确指定密钥路径，这样无论是否使用 ssh-add，Git 都能稳定工作

配置后，即使不手动执行 ssh-add，SSH 客户端也会通过 IdentityFile 找到密钥，且 AddKeysToAgent yes 会自动将密钥加载到代理（仅首次需要输入密码，之后缓存）。
